为什么Redis lua脚本具有原子性？
因为 Redis 的核心执行模型是「单线程事件循环」：
- 所有客户端的命令请求，都会进入一个 “命令队列”；
- Redis 主线程会依次从队列中取出命令执行，**同一时间只会执行一个命令**，不会有多个命令并行执行；
- 而 Redis 会把整个 Lua 脚本，视为「一个独立的、不可拆分的命令」
- 执行期间 “禁止中断”，要么全成要么全败
